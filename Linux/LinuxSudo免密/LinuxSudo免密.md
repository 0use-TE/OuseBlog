# Linux sudo 免密配置

每次敲 `sudo` 都要输入密码，有没有觉得烦？尤其是那种写着写着代码，突然要临时提权，结果还要停下来等密码输入的节奏感，真的很打断思路。

好消息是，你可以让指定用户sudo时不用输入密码。来看怎么做：

## 第一步：打开超级权限配置文件

```bash
sudo visudo
```

输入后系统会最后一次要求你输入当前用户的密码，验证通过后会打开一个类似 Nano 的文本编辑界面。

## 第二步：添加免密规则

用方向键一直拉到文件最底部，找到 `%sudo` 或 `User privilege specification` 那块区域。

假设你的用户名是 `ouse`，在文件最后一行添加：

```
ouse ALL=(ALL) NOPASSWD: ALL
```

这句话翻译成人话就是：**允许 ouse 这个用户，在任何地方以任何身份执行任何命令，并且不需要密码。**

## 第三步：保存并退出

如果你看到底部有 `^O Write Out` 的提示（说明是 Nano 编辑器）：

- 按 `Ctrl + O` 保存
- 按 `Enter` 确认文件名
- 按 `Ctrl + X` 退出

## 第四步：见证奇迹

现在随便敲一个需要权限的命令试试：

```bash
sudo apt update
```

你会发现系统直接放行，再也不会索要密码了，畅通无阻！

> ⚠️ **温馨提示**：这个配置适合个人开发机使用。如果是共享服务器，建议谨慎考虑安全风险。
